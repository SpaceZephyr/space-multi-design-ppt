# 内容分析与大纲规范

## 1. 内容分析（写大纲前完成）

- **核心信息**：一句话（≤15 字最好）说清"观众只记住一件事，是什么"
- **支撑点**：最多 3-5 个，按受众相关性排序（不是按原文顺序）
- **行动号召**：看完后希望观众做什么

### 受众适配

| 受众 | 内容侧重 | 视觉处理 |
|---|---|---|
| 高管 | 结论、ROI、战略影响 | 高度概括、干净、突出数据 |
| 技术 | 架构、实现、规格 | 详细图示、代码、示意图 |
| 大众 | 收益、故事、共鸣 | 视觉隐喻、简单图表 |
| 投资人 | 市场规模、增长、团队 | 增长曲线、里程碑、对比 |
| 学习者 | 步骤、示例、练习 | 渐进展开、编号步骤 |

### 内容取舍

- **保留**：核心论点、独家数据、受众相关案例、金句
- **简化**：技术细节→图示；长解释→层级要点；多案例→留最好的 1-2 个
- **可视化**：数据表→图表/大数字；流程描述→流程图；文字对比→左右对照
- **删除**：跑题信息、冗余案例、观众已知的背景

## 2. 大纲格式（outline.md）

```markdown
# {deck 标题}
风格: {品牌} | 受众: {类型} | 页数: {N} | 语言: {lang}

## 01 - 封面 [title-hero]
标题: {叙事式主标题}
副标题: {一句话}

## 02 - {slug} [布局]
标题: {叙事式标题}
要点: ...
视觉: {图表类型 / 大数字 / 对比 …}

...

## NN - 封底 [closing]
{行动号召 / 记忆点}
```

## 3. 叙事结构

- 开场（1-3 页）：钩子（惊人数据/提问/故事）→ 为什么现在重要 → 预告
- 中段模式任选其一：问题→方案 ／ 现状→冲突→解决 ／ What→Why→How ／ 过去→现在→未来 ／ 论点→证据→推论
- 收尾（1-2 页）：回扣核心信息 + 行动号召

## 4. 文案规则

1. **标题讲结论**："用量 6 个月翻倍" ✓，"关键数据" ✗；"一个平台替代五个工具" ✓，"我们的方案" ✗
2. **数据带来源**：页面角落 13px 标注出处
3. **禁 AI 套话**："让我们一起探索/深入了解"、"激动人心"、"总而言之" 一律不用；语气直接、自信
4. **封底有内容**：不是"谢谢观看"，而是明确的下一步、行动号召或一句记忆点
5. **无占位符**：每个元素写完整，不出现 "[待补充]"、"TBD"
